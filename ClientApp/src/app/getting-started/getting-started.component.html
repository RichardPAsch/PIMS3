- Initializing P.I.M.S. for first time use -
In order to efficiently, and as quickly as possible, setup this system for use, it is recommended that these steps be followed in the logical order presented; this may help address possible questions/concerns regarding menu or general P.I.M.S. functionality.
  1. 'Registration' - if not already done so, sign up by creating an account. Your entered 'Username' will serve as your login.
  2. 'Home - 'Login'' - Login using your prior Registration info. Upon a successful login, you will automatically be directed to the 'Income summary' page, which at this point will NOT yet contain any data. This is as expected.
  3. 'Data import'' - Consists of two parts for populating the system: 1) inserting assets/positions, and 2) revenue, both of which are available as selectable options, and require the use of existing templated XLSX spreadsheets. All columns are required, and should be populated as necessary. Both parts require proper file paths for import to work successfully. Asset/positions depend upon valid ticker symbols for fetching Profile data behind the scenes. Both revenue & position entries may be back-dated. Any problem tickers are most likely due to Profile generation, and may be resolved by skipping to 'Asset Profile' menu option.
    • Assets/Positions - Recover 'PositionsTemplate.xlsx' and populate with your portfolio holdings. Any problems, e.g., unkown tickers, accounts, etc., will be flagged and shown at the end of the import process. Unkown ticker symbols may be handled via the 'Asset profile' menu option. Positions marked with a status of "I" are inactive & represent assets that have been sold, e.g., called, or sold.
    • Revenue - Recover 'RevenueTemplate.xlsx' and populate with your position earnings. Actual importing of this type data should be deferred until ALL Position data has been first verified as Ok via the 'Positions' menu option!
  4. 'Positions' - Presents ( and confirms) existing Positions for your portfolio via a grid. This page allows for limited column editing/updates as well as sorting. Once confirmed all data entries are complete, proceed with executing 'Data import (revenue)' as noted from the step above.
  5. 'Asset profile' - Provides asset Profile information, serving the following multiple purposes:
    • Allows for a quick check (via web) of an Asset, returning dividend frequency, rate, etc. via 'Get Profile'
    • Allows for creation of a customized Profile, via 'Create Profile', as may be necessary if 'Get Profile' is either unable to fetch any data for an entered ticker, or if returned data is incomplete. A customized Profile is unique to each investor and may be updated, via 'Update Profile' as needed.
    • Allows for a quick check (via local storage) or update of a custom Asset via 'Get Saved Profile'
  6. 'Income recorded' - Presents all existing revenue saved. Income displayed may be filtered and/or sorted, as desired, and may date back to the last 5 years.
  7. 'Income summary' - Presents existing revenue saved, in summary form, for up to the last 8 years. Some columns are available for filtering and/or sorting.
  8. 'Income projections' - Allows for input of up to 5 different ticker symbols, as an an aid in determining projected monthly revenue. Optionally, both 'Price' and 'Rate' columns allow for manual edits, should your input data be more current.
